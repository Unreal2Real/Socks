# 项目规则

## 每次修改后必须提交

对本项目的任何代码修改（包括新建、编辑、删除文件）完成后，**必须立即执行 git 提交**，不得延迟积累。

### 提交规范

```bash
git add -A
git commit -m "type: 简短的修改说明"
git push
```

### Commit Message 格式

| type | 使用场景 |
|------|---------|
| feat | 新功能 |
| fix | 修复 bug |
| refactor | 重构代码 |
| docs | 文档变更 |
| style | 格式变化（不影响功能） |
| chore | 杂项（配置、依赖等） |

### 提交原则

- **小步提交**：每次修改独立提交，不要攒多个改动一起提
- **描述清晰**：用中文写明改了啥、为什么改
- **先提交再通知**：完成修改后先 commit + push，再告诉用户

## 禁止事项

- 不要留未提交的修改
