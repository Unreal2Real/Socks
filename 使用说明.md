# 「厂」字形态扫描工具 — 使用说明

## 什么是"厂"字形态？

"厂"字形态（类似杯柄形态 Cup-with-Handle）是一种技术分析形态，形如汉字"厂"：

```
     ┌──────────┐
    ╱            ╲
   ╱   上涨段     ╲──────────  横盘整理
  ╱                ╲
 ╱                  ╲
```

识别条件：
1. **上涨段** — 均线多头排列，涨幅 ≥ 15%
2. **横盘整理** — 布林带收窄、波动率降低、成交量萎缩（10~40天）
3. **形态完成** — 盘整结束后放量突破，形成买入信号

## 三、scan — 实时扫描

### 用法

```bash
# 增量扫描（默认模式）
# 只扫描"需要复查"的股票，日常使用
python3 main.py scan

# 全量扫描（扫描所有股票）
# 首次运行或每周做一次全量扫描
python3 main.py scan --full

# 限制扫描数量（调试用）
python3 main.py scan --limit 100

# 指定并行进程数
python3 main.py scan --workers 4

# 指定结果输出目录
python3 main.py scan --output my_results
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--full` | 关闭 | 全量模式，扫描所有股票 |
| `--limit N` | 无限制 | 只扫前N只（跳过指数后，索引210起才是实际股票） |
| `--workers N` | 3 | 并行进程数，CPU核数多可调大 |
| `--output DIR` | `results/` | 结果文件输出目录 |

### 扫描流程

```
获取股票列表（baostock）
       ↓
  过滤 ST 股、低价股、低成交量股
       ↓
  计算技术指标（MA、布林带、成交量比）
       ↓
  识别"厂"字形态
       ↓
  未匹配 → 判断是否接近形态（上涨中/盘整中）→ 存入 SQLite
  已匹配 → 存入结果文件，生成 K 线图
       ↓
  自动生成 HTML 报告（report/index.html）
```

### 输出文件

扫描完成后生成：
- `results/patterns_YYYYMMDD_HHMMSS.json` — 匹配形态的详细数据
- `results/patterns_YYYYMMDD_HHMMSS.csv` — CSV 格式
- `report/index.html` — 可视化的 HTML 报告
- `report/charts/*.png` — 每只匹配股票的 K 线图
- `report/grid_overview.png` — 所有形态的总览网格图

### 增量扫描机制

每只股票扫描后会分配复查间隔：

| 状态 | 含义 | 下次复查 |
|------|------|---------|
| `no_pattern` | 无明显形态 | 7天后 |
| `watching_uptrend` | 上涨中（涨幅不足15%） | 3天后 |
| `watching_consolidation` | 盘整中（接近形态） | 每天 |
| `matched` | 已匹配形态 | 5天后 |
| `error` | 数据异常 | 明天 |

> 增量模式下每天只扫几十~几百只股票，而不是全市场5000多只。

## 四、backtest — 历史回测

### 用法

```bash
# 指定时间段 + 股票数量回测
python3 main.py backtest --start 2025-06-01 --end 2026-04-29 --limit 200

# 指定具体股票回测
python3 main.py backtest --stocks 603135,301439,600054

# 从扫描结果文件加载股票列表
python3 main.py backtest --from results/patterns_20260429_171411.json

# 调整检查频率
python3 main.py backtest --start 2025-01-01 --end 2026-04-29 --stride 10
```

### 参数说明

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `--start DATE` | 120天前 | 回测起始日期（YYYY-MM-DD） |
| `--end DATE` | 今天 | 回测结束日期（YYYY-MM-DD） |
| `--stocks CODES` | 全市场 | 指定股票代码，逗号分隔 |
| `--from FILE` | 无 | 从扫描结果JSON加载股票 |
| `--stride N` | 5 | 每隔N个交易日检查一次 |
| `--data-days N` | 250 | 每次检查用多少天数据 |
| `--limit N` | 无限制 | 限制回测股票数量 |
| `--workers N` | 3 | 并行进程数 |
| `--output DIR` | `results/` | 输出目录 |

### 注意事项

- 回测时每个检查点往前取250天数据进行分析
- 步长（stride）越小检查越密集，但耗时越长
- 未指定股票时默认回测全市场（耗时较长，建议先 --limit）

## 六、配置文件说明

编辑 `config/settings.py` 可调整形态识别参数：

### 形态参数

```python
FACTORY_PATTERN_CONFIG = {
    'uptrend_gain': 0.15,                # 上涨段最少涨幅 (15%)
    'consolidation_days_min': 10,         # 盘整最少天数
    'consolidation_days_max': 40,         # 盘整最大天数
    'bandwidth': 0.20,                    # 布林带带宽上限
    'volatility': 0.20,                   # 波动率上限
    'volume_ratio': 0.6,                  # 成交量比上限（缩量要求）
    'ma_periods': [5, 10, 20, 60],       # 均线周期
    'bb_period': 20,                      # 布林带周期
    'bb_std_dev': 2,                      # 布林带标准差倍数
    'volume_ma_period': 20,               # 成交量均线周期
    'uptrend_min_days': 5,                # 上涨段最少天数
}
```

### 交易配置

```python
TRADING_CONFIG = {
    'market': 'A股',
    'data_days': 250,          # 每次分析取多少天数据
    'output_dir': 'results',   # 输出目录
}
```

### 股票过滤条件（`filters/stock_filter.py`）

| 条件 | 默认值 | 说明 |
|------|--------|------|
| 最低股价 | 1.0元 | 低于此价格的股票跳过 |
| 最高股价 | 500.0元 | 高于此价格的股票跳过 |
| 最低成交量 | 100万股 | 日均成交量低于此的跳过 |
| ST过滤 | 开启 | 自动跳过ST、\*ST股票 |

## 八、常见问题

### Q: 前几百只都是指数，怎么跳过？
A: baostock 返回的股票列表按代码排序，`000xxx` 开头的都是指数（约210只），实际股票从索引210开始。`scan --limit` 会包含指数，建议用 `--limit` 时取到 300+。

### Q: 为什么扫描没找到匹配？
A: 当前参数（盘整 max=40天）偏严格。很多股票盘整超过40天才出现特征。可调整 `consolidation_days_max` 或放宽 bandwidth/volatility 阈值。

### Q: 数据源是什么？需不需要付费？
A: 数据源为 [baostock](http://baostock.com)，免费开源，无需付费。

### Q: 扫描速度怎么样？
A: 增量模式扫几十只只需几秒。全量扫5000只需几分钟（取决于 workers 数量）。
