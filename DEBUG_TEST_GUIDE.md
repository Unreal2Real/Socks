## ✅ 已修复的问题清单（v2.2.0）

| 问题 | 严重程度 | 修复内容 |
|------|----------|----------|
| scanMode select 缺少 `id` 属性 | **严重** | 添加 `id="scanMode"`，增量/全量模式正确切换 |
| 进度显示使用全市场总数(3110) | **严重** | 改为使用实际扫描数量 |
| 股票列表接口卡死 | **严重** | 添加 `socket.setdefaulttimeout(15)` + try/except 错误处理 |

| Flask 单线程模式阻塞其他请求 | **严重** | 启用 `threaded=True` |
| 空 `except: pass` 隐藏错误 | **中等** | 添加 `print()` 日志输出异常信息 |
| 日志颜色区分不清晰 | **低** | info/progress 灰色、match/success 绿色、warn 橙色、error 红色 |
| 全量扫描 3110 只耗时过长导致前端超时 | **中等** | 添加 SSE 心跳保活 + 线程模式 |

## 🐛 常见问题排查

### 问题 1：HTTP 500 错误

**现象：**
```
[14:06:40] ✗ 扫描失败: HTTP 500
```

**排查步骤：**
1. **查看服务器终端日志**：寻找 Python 堆栈跟踪
2. **常见原因：**
   - 请求上下文错误 → 已在 v2.2.0 修复
   - 缺少 `_init_quotes_client` 方法 → 已在 v2.2.0 修复
   - mootdx 未安装 → `pip install mootdx`

---

### 问题 2：扫描进度不更新

**现象：**
- 进度长时间卡在某个百分比

**排查步骤：**
1. 打开浏览器控制台（F12 → Console）看是否有错误
2. 查看服务器终端是否有异常输出
3. 检查是否该股票的数据获取卡死（socket 超时已设为 15 秒）

---

### 问题 3：K线图加载失败

**现象：**
```
K线图 加载失败：network error
```

**排查步骤：**
1. 单独测试：
   ```bash
   curl "http://localhost:5555/api/stock/daily?code=600519&days=60"
   ```
2. 该接口不受 mootdx 依赖影响，只依赖 Baostock
3. 检查网络连接

---

### 问题 4：分时数据加载失败

**现象：**
```
获取分时数据失败
```

**排查步骤：**
1. 该接口依赖 `mootdx` 库：
   ```bash
   pip install mootdx
   ```
2. 检查 `config/settings.py` 中 `tdx_path` 路径是否正确
3. 确保通达信软件已启动并更新了数据

---

### 问题 5：扫描停止后 CPU 仍然高

**现象：**
- 点击「停止扫描」后服务器 CPU 仍然占用高

**测试方法：**
1. 开启任务管理器
2. 开始扫描，然后点击停止
3. 观察 CPU 是否在 2 秒内回落

**如果仍然高：**
- 重启 API 服务器
- 检查 `api_server.py` 中 `threaded=True` 是否启用
